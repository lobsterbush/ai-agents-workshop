Part 1

How Agents Work

Agents have access to tools

File System

Read, write, organize any file on your machine

Terminal

Execute commands, run scripts, install packages

Web Access

Search documentation, fetch resources, verify facts

Code Execution

Run R, Python, Stata—see results, fix errors

The loop

Agents don't respond once. They observe each result, then decide what to do next.

Observe
→
Plan
→
Act
→
Evaluate
→
Repeat

This is why they recover from errors. They see what went wrong and try again.

Part 3

Where They Fail

Known failure modes

Hallucinated citations. They invent plausible-sounding references. Every citation must be verified.

Confident but wrong statistics. Numbers can be subtly incorrect. Always check the math.

Overfit to your prompt. They do exactly what you ask, even if it's wrong to ask.

The verification principle

Treat agent output like a research assistant's first draft.

  • Run the code yourself
  • Spot-check calculations
  • Verify every citation exists
  • Review the logic, not just the output

Three things

  • Verify everything. Trust requires validation.
  • Document by default. The log is your reproducibility.

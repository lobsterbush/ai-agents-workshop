Key Concept

You interact with and direct the agent

This isn't fire-and-forget. You're in a conversation.

  • Give instructions — describe what you want in plain language
  • Watch it work — see every command, every file change in real time
  • Intervene — correct course, add context, change direction mid-task
  • Iterate — refine the output through follow-up prompts

Think of it as pair programming with a very fast junior colleague. You set direction. It executes. You review.

Part 1

How Agents Work

Agents have tools

They can interact with your computer the same way you do.

  • File system — read, write, organize, delete
  • Terminal — run commands, install packages, execute scripts
  • Code execution — run R, Python, Stata; see errors; fix them
  • Web access — search documentation, fetch data, verify information

The loop

Agents observe each result and decide what to do next.

  1. Observe current state
  2. Plan next action
  3. Execute action
  4. Check if it worked
  5. Repeat until done

This is why they recover from errors. They see what went wrong and try again.

What matters

Every command is logged. Every file change is tracked. The workflow is reproducible by default.

This isn't just faster. It's a different way of working—where documentation happens automatically.

Reproducibility

Built-in replication

Warp creates an audit trail without any extra effort.

  • Full session logs — every prompt, every command, every output is recorded
  • Model tracked per interaction — which LLM (Sonnet, Opus, GPT-4o) produced each result
  • Export sessions — save complete interaction history as shareable files
  • Pair with git — commits + session logs = complete provenance chain

Save your Warp session logs alongside your data and code. Reviewers can see exactly how results were produced—including which model generated each step.

Part 3

Where They Fail

Known failure modes

These tools are powerful but not infallible.

  • Hallucinated citations — they invent plausible-sounding references that don't exist
  • Wrong statistics — numbers can be subtly incorrect; always verify
  • Overfit to prompt — they do exactly what you ask, even if it's the wrong thing to ask
  • Confident errors — mistakes are presented with the same confidence as correct output

The verification principle

Treat agent output like a research assistant's first draft.

  • Run the code yourself
  • Spot-check calculations against known values
  • Verify every citation exists (Google Scholar, library)
  • Review the logic, not just the output
  • Check edge cases and assumptions

Best Practice

Prototype fast, iterate faster

Agents excel at rapid prototyping. Use this strategically.

  • First draft in seconds — get a working version, then refine through conversation
  • Try multiple approaches — "Now try this with a different specification" costs nothing
  • Fail fast — test an idea before committing hours to it
  • Robustness checks — "Run this with every combination of controls" in one prompt

The cost of trying something has collapsed. The cost of not trying just went up.

Prompt Card 2

Quantitative Analysis

Re-analyzing old, forgotten, or reused data.

$ "I have a dataset at [FILE PATH] that I collected a while
ago but never fully analyzed.

1. Explore it — tell me what's in it, how many observations,
   key variables
2. Run [MODEL TYPE] predicting [DV] with [IVs] using
   robust standard errors
3. Create publication-ready figures using ggplot with
   theme_tufte()
4. Generate a LaTeX results table
5. Write a results paragraph with inline statistics
6. Put all output in a new analysis/ folder"

Prompt Card 3

Survey Instrument

Creating a survey with validated scales and experimental treatments.

$ "Design a survey experiment on [TOPIC YOU CARE ABOUT].

1. Find validated scales for measuring [KEY CONSTRUCTS]
2. Create experimental treatments that vary [MANIPULATION]
3. Write survey items with proper response scales
4. Add attention checks and demographic questions
5. Randomize treatment assignment
6. Format everything for import into Qualtrics
7. Write a brief pre-registration document

⚠️ VERIFY all scale citations — agents hallucinate"

Choose your challenge

Use one of the prompt cards above, or make up your own.

  • Starter: Use Prompt Card 1 — navigate to a real project directory and have the agent map it out
  • Standard: Use Prompt Card 2 — point at a real dataset and get a full analysis pipeline
  • Advanced: Use Prompt Card 3 — design a complete survey experiment on a topic you care about

Pick the one that slightly scares you.

Part 5

Implications

The question isn't whether

These tools are here. They're getting better rapidly. The question is how we use them responsibly as a scholarly community.

Equilibrium

What AI agents drive down

The cost of execution is collapsing across the research pipeline.

↓ Data collection
Scraping, surveys, API calls, simulation
↓ Data cleaning
Parsing, merging, imputation, validation
↓ Data analysis
Models, robustness checks, visualizations
↓ Results presentation
Tables, figures, LaTeX, web apps
↓ Writing
Drafts, lit reviews, technical prose
↓ Complex environments
Web apps, virtual worlds, interactive experiments

Equilibrium

What gains value

When execution is cheap, what's scarce becomes more valuable.

  • ↑ Ideas — original questions, creative research design, theoretical insight
  • ↑ Taste — knowing what's worth doing, what's interesting, what's important
  • ↑ Fieldwork — ethnography, interviews, participant observation
  • ↑ Field experiments — real-world interventions that require human presence
  • ↑ Verified human involvement — anything that requires being there, in person, in context

The new premium

In a world where an agent can clean your data, run your models, and draft your paper overnight—

the researcher who has the idea, who spent six months in the field, who has the judgment to know what matters

becomes more valuable, not less.

Discussion

  • What research tasks would benefit most from agent assistance?
  • Where should human judgment remain non-negotiable?
  • What norms should we adopt for transparency and acknowledgment?
  • How do we train the next generation of researchers for this world?
